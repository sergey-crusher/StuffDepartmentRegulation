# Список Документов

Вопрос разработки документации по ГОСТ больше в том — **на сколько полной должна быть формализация** проекта и хватит ли на это ресурсов?

Постараемся определить здесь, какая последовательность документов должна быть соблюдена в зависимости от **этапов** производственного процесса.

1. Предпроектное исследование
2. Техническое задание
3. Эскизный проект
4. Технический проект
5. Разработка рабочей документации

Спасибо за полезную информацию. Не соглашусь только с тем, что эскизный проект это упрощённая версия технического. **На эскизнике, обычно, рассматриваются варианты технических решений** и принимаются принципиальные решения. На техпроекте уже детально прорабатываются окончательные технические решения.

На эскизном проекте мы разрабатываем от 1 (П1) до 4 документов (П1, С1, С2, СО) по ГОСТ 34.201–2020. (источник https://habr.com/ru/companies/angarasecurity/articles/671882/)

## Какими ГОСТами регулируется

- **ГОСТ Р 59795-2021** «Комплекс стандартов на автоматизированные системы. Автоматизированные системы. Требования к содержанию документов» – для определения содержания документа.
    
- **ГОСТ 34.601-90** «Комплекс стандартов на автоматизированные системы. Автоматизированные системы. Стадии создания» – определяет на какой стадии создания АС какой документ должен быть разработан, а в нашем случае документ «Общее описание системы».
    
- **ГОСТ 34.201-2020** «Комплекс стандартов на автоматизированные системы. Виды, комплектность и обозначение документов при создании автоматизированных систем».
