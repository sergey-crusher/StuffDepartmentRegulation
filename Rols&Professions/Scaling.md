### 🟢 Минимальный набор (Стартап / Малая команда < 10 человек)

**Фокус: «Максимальное совмещение ролей для выживания и скорости»**

В этой модели все роли жёстко совмещены, а границы ответственности размыты.

| Совмещённая роль                                  | Ядро ответственности (Что нельзя делегировать)                                                                                                                                | Ключевые выходы / Артефакты                                                                                  |
| ------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| **Tech Lead + Tech Owner + Архитектор**           | • Принятие окончательных технических решений  <br>• Ответ на вопрос "Как это сделать?" перед командой и "Возможно ли это?" перед клиентом  <br>• Выбор технологического стека | • Техническое видение продукта  <br>• Архитектурные схемы  <br>• Оценки сложности и рисков                   |
| **Product Manager + Project Manager**             | • Ответ на вопрос "Что строить и почему?"  <br>• Формирование дорожной карты (Roadmap)  <br>• Контроль сроков и приоритетов                                                   | • Бэклог продукта  <br>• Roadmap  <br>• Отчёты о статусе проекта                                             |
| **Engineering Manager + Team Lead**               | • Найм/увольнение  <br>• Ежедневное распределение задач и менторство  <br>• Мотивация команды и разрешение конфликтов                                                         | • Сформированная и работоспособная команда  <br>• План развития сотрудников  <br>• Регулярная обратная связь |
| **Универсальный разработчик (Dev + QA + DevOps)** | • Качество и работоспособность своего кода  <br>• Деплой и базовый мониторинг                                                                                                 | • Рабочий код в проде  <br>• Набор автоматических тестов                                                     |

**Главный риск:** "Узкое горлышко" в лице Tech Lead/Tech Owner. Его перегруженность — главная угроза проекту.

### 🟡 Нормальный набор (Растущая компания / Несколько команд)

**Фокус: «Разделение стратегической и тактической работы для качества и масштабируемости»**

Происходит разделение ролей по зонам ответственности: продукт, проект, люди, технологии.

| Роль                                     | Ключевая ответственность                                                                                                                                                                                             | С кем взаимодействует постоянно                                                                              |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| **Tech Owner / Technical Product Owner** | **"Что" (техническая часть):** Техническое видение продукта, управление техническим бэклогом (техдолг, инфраструктура), приемка сложных фич. **Ответ на вопрос клиента: "Реализуемо ли это и как устроено внутри?"** | • Product Manager (стратегия)  <br>• Solution Architect (решение)  <br>• Команда разработки (реализация)     |
| **Solution Architect**                   | **"Как в рамках проекта":** Проектирование целостного решения под задачи клиента, выбор компонентов, описание интеграций. Несёт ответственность за архитектурную целостность.                                        | • Tech Owner (ограничения продукта)  <br>• Клиент/Заказчик (требования)  <br>• Development Team (реализация) |
| **System Architect**                     | **"Как в рамках платформы/компании":** Стратегия развития технологической платформы, стандарты кодирования, инструменты разработки.                                                                                  | • Tech Owner, Solution Architect  <br>• Руководство компании (техстратегия)                                  |
| **Product Manager**                      | **"Что" (бизнес-часть) и "Зачем":** Бизнес-ценность, метрики, рынок, Roadmap.                                                                                                                                        | • Tech Owner (технические возможности)  <br>• Project Manager (сроки)  <br>• Бизнес/Клиент                   |
| **Project Manager**                      | **"Когда" и "Сколько":** Сроки, бюджет, ресурсы, риски, коммуникация.                                                                                                                                                | • Product Manager (приоритеты)  <br>• Engineering Manager (ресурсы)  <br>• Команда (прогресс)                |
| **Engineering Manager**                  | **"Кто":** Найм, развитие, мотивация, карьеры, зарплаты. Отвечает за команду как ресурс.                                                                                                                             | • Team Lead (ежедневная работа команды)  <br>• Project Manager (потребности в ресурсах)  <br>• HR            |
| **Team Lead**                            | **"Как сегодня":** Ежедневное ведение команды, распределение задач, код-ревью, решение операционных проблем.                                                                                                         | • Команда разработки  <br>• Engineering Manager (развитие людей)  <br>• Tech Owner (технические задачи)      |
| Developer                                |                                                                                                                                                                                                                      |                                                                                                              |

**Преимущество:** Снижается нагрузка на ключевых специалистов, решения становятся более качественными.

### 🔴 Максимальный набор (Крупный enterprise, распределённые команды)

**Фокус: «Глубокая специализация, управление сложностью и стратегией»**

Появляются нишевые роли, отвечающие за отдельные аспекты на высоком уровне абстракции.

| Роль                                                          | Ключевая ответственность                                                                                                    | В какой поток встраивается                                         |
| ------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| **Enterprise Architect**                                      | **IT-стратегия компании:** Согласованность всех IT-систем между собой и с бизнес-целями. Выбор ключевых платформ.           | • Стратегический комитет  <br>• Бизнес-архитекторы                 |
| **Chief Technology Officer (CTO)**                            | **Технологическая стратегия компании в целом:** Инвестиции в технологии, R&D, общее техническое видение.                    | • Руководство компании  <br>• Продуктовые и архитектурные комитеты |
| **Technical Program Manager**                                 | **Координация кросс-функциональных инициатив:** Управление несколькими связанными проектами, зависимостями между командами. | • Менеджеры проектов и продуктов  <br>• Архитекторы                |
| **Principal Engineer / Lead Developer**                       | **Глубокая техническая экспертиза и инновации:** Решение самых сложных технических проблем, R&D, эталонное качество кода.   | • Работает across командами  <br>• Tech Owner, System Architect    |
| **Product Owner**                                             | **Тактическое управление бэклогом одной команды:** Максимизация ценности работы своей команды на тактическом горизонте.     | • Product Manager (стратегия)  <br>• Development Team (исполнение) |
| **Бизнес-аналитик, Data Architect, Security Architect и др.** | **Глубокая экспертиза в узкой области:** Детализация требований, проектирование данных, безопасность.                       | • Соответствующие потоки и проекты                                 |

### 💡 Итог: Кто говорит с клиентом о реализуемости?

- **В минимальной модели:** **Tech Lead/Tech Owner** — это единственный человек, кто технически подкован для этого диалога.
    
- **В нормальной модели:** Происходит разделение:
    
    - **Solution Architect** проектирует решение, исходя из потребностей клиента.
        
    - **Tech Owner** говорит о реализуемости с точки зрения возможностей продукта и команды.
        
    - Вместе они дают клиенту полную и непротиворечивую картину.
        
- **В максимальной модели:** В диалоге с клиентом могут участвовать **Solution Architect, Technical Program Manager** и даже **Enterprise Architect**, в зависимости от масштаба и стратегической важности проекта.
